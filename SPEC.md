## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 個人電商賣家 | ~10 萬 | 重複回覆私訊、整理訂單、生成週報，每天耗 2-3 小時 |
| 小型服務業（美髮/餐飲/工作室） | ~15 萬 | 客戶預約、表單回覆、行銷素材製作，無專人處理 |
| KOL / 自媒體小編 | ~5 萬 | 每天要發 2-3 Reels、回留言、整理數據，無行銷助理 |
| 個人接案 / SOHO 族 | ~30 萬 | 報價、合約、發票、客戶溝通，一人分飾多角 |

### 1.2 為什麼不做替代方案

- **真人外包（接案平台）**：每月 15,000-50,000 NT$，對微型賣家太貴
- **商用 RPA（UiPath/Automation Anywhere）**：年費 30 萬起，複雜度過高
- **現有 AI 工具（單一功能）**：回覆用 ChatGPT、設計用 Canva、報表用 Excel — 切換多個工具反而更沒效率
- **我們的解法**：把 144 種真實職位（agency-agents 啟發）轉成 AI Agent，1 個月費 99 USD 起，多 Agent 協作

## 2. 解決方案

### 2.1 核心價值主張

> 「每月 NT$2,990 請 5 個 AI 員工 — 私訊小編、訂單助理、報表分析師、客服專員、行銷企劃。」

### 2.2 使用者流程

1. 選擇要僱用的 AI 員工（5 種熱門預載 + 144 種完整目錄）
2. 每個 AI 員工有獨立任務 inbox（待辦 / 進行中 / 完成）
3. 設定觸發條件（例：FB 私訊 → AI 客服自動回覆）
4. AI 員工執行任務，使用者可審核 / 修改 / 核准
5. 月底 AI 員工自動生成工作報表（節省時間 + 數據）

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 5 種預載 AI 員工（私訊小編/訂單助理/客服/行銷企劃/報表分析師）
- [ ] 144 種完整 AI 員工目錄（從 agency-agents 啟發）
- [ ] 每個 AI 員工獨立任務 inbox（待辦/進行中/完成）
- [ ] 觸發條件設定（FB 私訊 / 訂單通知 / Email / 排程）
- [ ] AI 任務執行（使用 GPT-4 / Claude API）
- [ ] 人類審核機制（AI 產出需人工核准才送出）
- [ ] 月底工作報表

### 3.2 v2（加值）

- [ ] 多 AI 員工協作（任務派工）
- [ ] 自訂 AI 員工（訓練專屬 SOP）
- [ ] 整合 LINE / IG / FB Messenger API
- [ ] 客戶滿意度分析

### 3.3 明確不做

- 完全無人監督模式（堅持人類審核）
- 法律 / 醫療專業建議（AI 員工不做）
- 跨境支付（先做台灣市場）
- 真人客服轉接（純 AI Agent）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | Next.js 14 + TypeScript | SSR 強、AI streaming UX 友善 |
| 後端 | Next.js API Routes + FastAPI | 任務 queue 需 async |
| LLM | OpenAI GPT-4 + Claude 3 | 主力 + 備援 |
| 任務佇列 | BullMQ + Redis | 背景任務處理 |
| 資料庫 | PostgreSQL + Prisma | 多用戶、多任務 |
| 部署 | Vercel + Railway |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| LLM 成本超支（高頻使用） | 🔴 高 | 用量配額、計價模式（每次任務計算 token）、成本監控 dashboard |
| AI 產出品質不穩 | 🟠 中 | 人類審核機制 + 信心分數標示 |
| 多 AI 員工協作衝突 | 🟠 中 | 明確任務邊界 + 衝突偵測 |
| 用戶隱私資料（客戶私訊） | 🟠 中 | 加密儲存 + GDPR 個資處理流程 |
